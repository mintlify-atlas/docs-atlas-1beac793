title: "Get started with HaishinKit"
description: "Publish your first live stream with HaishinKit. Install the SDK, configure AVAudioSession, create a MediaMixer, attach camera and microphone, then connect via RTMP."
---

This guide walks you through setting up a complete RTMP publish stream on iOS using HaishinKit. You will install the package, configure the required system permissions, create a `MediaMixer` to manage camera and microphone inputs, and connect to an RTMP server to start publishing.

<Steps>
  <Step title="Install HaishinKit">
    Add HaishinKit to your Xcode project using Swift Package Manager.

    In Xcode, go to **File → Add Package Dependencies** and enter the repository URL:

    ```
    https://github.com/HaishinKit/HaishinKit.swift
    ```

    When prompted to choose package products, select **RTMPHaishinKit** (which automatically pulls in the **HaishinKit** core). See the [installation guide](/installation) for a full description of all available modules.
  </Step>

  <Step title="Add Info.plist permission keys">
    HaishinKit accesses the camera and microphone, so your app must declare usage descriptions in `Info.plist`. Add the following keys:

    ```xml
    <key>NSCameraUsageDescription</key>
    <string>your usage description here</string>
    <key>NSMicrophoneUsageDescription</key>
    <string>your usage description here</string>
    ```

    <Warning>
      Apple will reject your app from the App Store if these keys are missing. The strings are shown to users in the system permission dialog, so make them meaningful.
    </Warning>
  </Step>

  <Step title="Configure AVAudioSession">
    On iOS, you must configure and activate `AVAudioSession` before HaishinKit can capture audio. Call this early in your app's lifecycle — for example, in `application(_:didFinishLaunchingWithOptions:)` or your view's `onAppear`:

    ```swift
    import AVFoundation

    let session = AVAudioSession.sharedInstance()
    do {
        try session.setCategory(.playAndRecord, mode: .default, options: [.defaultToSpeaker, .allowBluetooth])
        try session.setActive(true)
    } catch {
        print(error)
    }
    ```

    <Note>
      The `.playAndRecord` category allows simultaneous capture and playback. `.allowBluetooth` enables Bluetooth headsets as an audio input source.
    </Note>
  </Step>

  <Step title="Create a MediaMixer and attach devices">
    `MediaMixer` is the core of HaishinKit's media pipeline. It manages camera and microphone inputs and routes mixed output to one or more stream connections.

    ```swift
    import HaishinKit
    import RTMPHaishinKit
    import AVFoundation

    let mixer = MediaMixer()
    let connection = RTMPConnection()
    let stream = RTMPStream(connection: connection)

    Task {
        do {
            // Attach the default microphone
            try await mixer.attachAudio(AVCaptureDevice.default(for: .audio))
        } catch {
            print(error)
        }

        do {
            // Attach the back wide-angle camera
            try await mixer.attachVideo(AVCaptureDevice.default(.builtInWideAngleCamera, for: .video, position: .back))
        } catch {
            print(error)
        }

        // Route mixed output to the RTMP stream
        await mixer.addOutput(stream)
    }
    ```
  </Step>

  <Step title="Display a live preview">
    Use `MTHKView` to render the stream's video in your view hierarchy. Add it to your view before connecting so you can see a live preview from the camera.

    ```swift
    let hkView = MTHKView(frame: view.bounds)

    Task { @MainActor in
        await stream.addOutput(hkView)
        // Add the view to your view controller's view
        view.addSubview(hkView)
    }
    ```
  </Step>

  <Step title="Connect and publish via RTMP">
    Connect to your RTMP server and begin publishing. Replace `localhost/appName/instanceName` with your server's address and `streamName` with your actual stream key.

    ```swift
    Task {
        do {
            try await connection.connect("rtmp://localhost/appName/instanceName")
            try await stream.publish(streamName)
        } catch RTMPConnection.Error.requestFailed(let response) {
            print(response)
        } catch RTMPStream.Error.requestFailed(let response) {
            print(response)
        } catch {
            print(error)
        }
    }
    ```

    <Tip>
      To use RTMPS (RTMP over TLS), replace `rtmp://` with `rtmps://` in the connection URL. No other changes are required.
    </Tip>
  </Step>
</Steps>

## Playback example

To play back an RTMP stream, create a connection and stream, attach an `AudioPlayer`, then call `play` instead of `publish`:

```swift
import HaishinKit
import RTMPHaishinKit
import AVFoundation

let connection = RTMPConnection()
let stream = RTMPStream(connection: connection)
let audioPlayer = AudioPlayer(AVAudioEngine())
let hkView = MTHKView(frame: view.bounds)

Task { @MainActor in
    await stream.addOutput(hkView)
}

Task {
    // AudioPlayer must be attached before connecting
    await stream.attachAudioPlayer(audioPlayer)

    do {
        try await connection.connect("rtmp://localhost/appName/instanceName")
        try await stream.play(streamName)
    } catch RTMPConnection.Error.requestFailed(let response) {
        print(response)
    } catch RTMPStream.Error.requestFailed(let response) {
        print(response)
    } catch {
        print(error)
    }
}
```

<Note>
  `AudioPlayer` wraps `AVAudioEngine` and must be attached to the stream before you call `connect`. If you skip this step, playback audio will be silent.
</Note>

## Connecting with authentication

For RTMP servers that require FMLE-compatible authentication, include credentials directly in the connection URL:

```swift
connection.connect("rtmp://username:password@localhost/appName/instanceName")
```
